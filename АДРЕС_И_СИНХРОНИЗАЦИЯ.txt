ППР Контроль v104-stable

Синхронизация ускорена:
- основной быстрый канал: /api/events через EventSource, без внешних пакетов;
- при изменении сервер сразу отправляет событие другим открытым пользователям;
- тест на локальном сервере показал доставку события примерно 75 мс;
- если быстрый канал недоступен, резервный опрос работает примерно каждые 2 секунды;
- отправка локальных изменений после ввода/действия уменьшена с 900 мс до 300 мс.

Для Render:
- тип сервиса: Web Service;
- Build Command: npm install;
- Start Command: npm start.

Важно: GitHub Pages как Static Site не подходит для общей синхронизации. Нужен Node.js сервер, например Render Web Service.
