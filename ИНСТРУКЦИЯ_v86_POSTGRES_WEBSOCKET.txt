V86: PostgreSQL + WebSocket

Что изменено:
1) Добавлен WebSocket (/ws): комментарии и заявки обновляются у других пользователей почти сразу.
2) Добавлена поддержка PostgreSQL через переменную DATABASE_URL.
3) Если DATABASE_URL не указан, проект временно работает по старому через data/db.json.
4) Добавлен /api/health для проверки: покажет storage: postgresql или db.json.

Как запускать на Render:
1) Загрузить все файлы из архива в GitHub.
2) Render → Web Service → Manual Deploy → Deploy latest commit.
3) В Render добавить Environment Variable:
   DATABASE_URL = строка подключения PostgreSQL
4) Открыть:
   https://твой-сайт.onrender.com/api/health
   Нужно увидеть: "storage":"postgresql", "websocket":true

Важно:
- Для 50 пользователей лучше платный VPS/Render и PostgreSQL.
- Без DATABASE_URL будет работать db.json fallback, но это не лучший вариант для 50 пользователей.
- Интервал опроса оставлен 10 секунд только как запасной режим. Основная синхронизация теперь через WebSocket.
